<p>Google made one of its biggest changes with how it ranks web pages this year. The update was rolled out in mid-June 2021 by introducing the Core Web Vitals.&nbsp; Any business that values brand awareness and visibility should care about it.</p>

<figure class="wp-block-image size-full"><img src="/images/blog/Screenshot-2024-02-15-at-11.01.21%E2%80%AFAM.png" alt="Core Web Vitals" class="wp-image-590"/></figure>

<p>Core Web Vitals are a set of user-focused metrics designed to measure a page’s content loading speed, interactivity and visual stability in terms of providing a smooth and seamless user experience.</p>

<p>So why do you care as a website owner or store owner about the Core Web Vitals? Because after all,&nbsp;<a href="https://videos.brightedge.com/research-report/BrightEdge_ChannelReport2019_FINAL.pdf">53% of all website traffic</a>&nbsp;comes from organic searches or search engine results that are naturally generated and not influenced by commercial relationships.&nbsp; A better user experience, equals higher traffic and more revenue.&nbsp;&nbsp;</p>

<p>Because Google is a search engine, and its job is to match a user’s searches with the best possible results.&nbsp; Google uses Core Web Vitals to rank your page vs other pages when deciding where your page should be displayed, whether in the first page or in the twentieth page.&nbsp;</p>

<p>Before the introduction of the Core Web Vitals, Google found the high-quality content that matched the search. But also, it would take into consideration whether the page was mobile-friendly, safe browsing and provided a good experience.</p>

<p>Now, event content will always be stellar for SEO (Search Engine Optimization), websites that fail to apply the best practices will be given a lower ranking score than ones that do.</p>

<p>With mobile traffic taking over regular desktop traffic, it is no wonder why Google is weighting the ranking of websites heavily if they provide a good Mobile Experience. So think mobile first when designing a website.</p>

<h3 class="wp-block-heading">Core Web Vitals</h3>

<p>Here are the Core Web Vitals that Google will be using to assess a website’s user experience score:</p>

<ul><li>LCP or (Largest Contentful Paint)</li>

<li>FID or (First Input Delay)</li>

<li>CLS or (Cumulative Layout Shift)</li>
</ul>

<figure class="wp-block-image size-full"><img src="/images/blog/Screenshot-2022-12-17-at-8.55.30-AM-1024x458-1.png" alt="" class="wp-image-589"/><figcaption class="wp-element-caption">LightHouse Report running in Chrome</figcaption></figure>

<p>Each Core Web Vital will be scored as either ‘Good’, ‘Needs Improvement’, or ‘Poor’.&nbsp; So website owners and store owners now know that a good user experience is no longer a “would be nice to have”, but an absolute “must-have”, as it’s critical for good SEO.&nbsp; A bad ranking will, in turn, damage their traffic, conversions, and ultimately, revenue.</p>

<h4 class="wp-block-heading">Largest Contentful Paint</h4>

<p>The&nbsp;<strong>LCP</strong>&nbsp;metric refers to the loading performance of a site, it measures how long it takes to load the largest content element on the page. This can be an image, video, animation, photo gallery, etc.&nbsp; If a page has some text, medium size images, and a video, the video would probably be the largest element on the page.</p>

<p>A LCP of 2.5 seconds or less scores ‘Good’, between 2.5 – 4.0 seconds scores ‘Needs Improvement’, and anything over 4.0 seconds scores ‘Poor’.&nbsp;&nbsp;</p>

<p>But in my opinion, any element that takes one second or more to load, ‘Needs Improvement’.&nbsp; Taking into consideration that 1 second delay results in a 7% drop in conversion\revenue.</p>

<h4 class="wp-block-heading">First Input Delay</h4>

<p>The&nbsp;<strong>FID</strong>&nbsp;metric is a test to measure the time it takes for a page to react to the user’s first interaction. &nbsp; A quicker response time leads to a better user experience. It measures mouse moves and clicks, trackpad taps, and key presses that require the browser to respond.&nbsp;</p>

<p>A FID score of anything under 100 ms scores ‘Good’, between 100 ms – 300 ms scores ‘Needs Improvement’, and anything above 300 ms scores ‘Poor’.</p>

<h4 class="wp-block-heading">Cumulative Layout Shift</h4>

<p>The&nbsp;<strong>CLS</strong>&nbsp;metric uses multiple frames of the page as it loads to determine when the elements shift and by how much.&nbsp;</p>

<p>A CLS of below 0.1 scores ‘Good’, anything between 0.1 and 0.25&nbsp; scores ‘Needs Improvement’, and anything over 0.25 scores ‘Poor’.</p>
